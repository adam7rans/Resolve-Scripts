# Dither Studio

Standalone tool that runs the **same animated dither‑noise background and the same
video dither pipeline** that live on the `w3rk17` portfolio site, and exports them
as a **PNG sequence** ready to drop into DaVinci Resolve.

Two views:

1. **Background** — the domain‑warped fbm/simplex/value/worley shader from
   `w3rk17/src/components/ThreeGradientBackground.tsx`, post‑processed by the
   same dither pass (`w3rk17/src/shaders/ditherShader.ts`).
2. **Talking video** — drag‑and‑drop a video, run it through the
   `talkingLevelsShader` (brightness/contrast/levels/gamma/zone control) and
   then the same dither post‑pass. Use this on recordings of yourself talking.

The shaders are copied verbatim from `w3rk17` into `src/shaders/` so the look
matches the site exactly.

## Run it

```bash
cd "resolve plugins/dither-studio"
npm install
npm run dev
```

Open the URL Vite prints (default `http://127.0.0.1:5180`).

> **Use Chrome / Edge / Brave / Arc.** Export uses the File System Access API.
> Safari and Firefox can preview but cannot write the PNG sequence.

## Workflow

Projects live in `../projects/<project-name>/` beside this app. Each project
folder contains:

- `project.json` — project identity plus imported media metadata.
- `settings.json` — the saved right-panel UI/shader/export settings.
- the imported video file.
- `caption.json` — generated by AssemblyAI or saved from a manually loaded
  caption/transcript JSON.

When you choose a video in the browser file picker, the app imports a copy into
the active project folder and uses that managed copy from then on. Browsers do
not expose the original Downloads path to web apps, so the original source file
cannot be deleted automatically from Downloads by this app.

### Background → Resolve

1. Tweak the noise + dither controls in the right panel until you like it.
2. In **Export**: set width / height / fps / duration, then click
   **Export PNG sequence**, pick an empty folder.
3. In Resolve: **Media Pool → Import Media** → select the **first** PNG → in
   the import dialog enable **"Image Sequence"** → drop on a timeline. Resolve
   will render at any project frame rate.

### Talking video → Resolve

1. Create or select a project.
2. Click **Choose video…** or drop a video file on the preview area.
3. The video appears immediately, is imported into the project folder, and the
   backend extracts audio and sends it to AssemblyAI. Progress appears in the
   status row and toast messages. When AssemblyAI returns captions, the app saves
   `caption.json` into the project folder and loads it automatically.
4. Adjust **Levels** (works like a curves/levels pre‑pass on the video) and
   **Dither** controls. Preview plays in real time.
5. **Export PNG sequence** — duration is locked to the video's duration; fps
   defaults to the video's native size. Export, then import into Resolve as an
   image sequence and replace your raw clip.

If your video has audio you want to keep, lay the original clip on a track
below the rendered sequence in Resolve and turn its video track off — the
sequence and the original audio will stay in sync because every PNG matches
exactly `frame_index / fps` seconds.

## Why PNG sequence?

- Lossless, alpha‑capable, deterministic (re-exports are byte-identical).
- Resolve treats image sequences as first‑class media; you can change project
  fps and Resolve retimes correctly.
- No external dependencies (no ffmpeg, no codec install).

## Files map (dither studio ↔ w3rk17)

| this app | w3rk17 source |
|---|---|
| `src/shaders/ditherShader.ts` | `w3rk17/src/shaders/ditherShader.ts` (copied) |
| `src/shaders/talkingLevelsShader.ts` | `w3rk17/src/shaders/talkingLevelsShader.ts` (copied) |
| `src/shaders/backgroundShader.ts` | extracted from `w3rk17/src/components/ThreeGradientBackground.tsx` |
| `src/lib/DitherPass.ts` | mirrors `w3rk17/src/components/three/PostProcessor.tsx` |
| `src/lib/BackgroundRenderer.ts` | vanilla rewrite of `ThreeGradientBackground.tsx`, with deterministic `setTime + renderFrame` |
| `src/lib/VideoRenderer.ts` | combines the patterns from `DitherVideoEffect.tsx` + `talkingLevelsShader.ts` |

If you tweak shader code on the site, copy the changed `*.ts` shader files
across to keep both in sync.
